# Academy Attendance & CRM Web App Development Rules

## 1. 배포 및 동기화 의무화 (Vercel Live Sync)
- 모든 수정 사항은 로컬 파일에만 적용해서는 안 되며, 반드시 Git 커밋 후 GitHub 원격 저장소(`origin main`)에 푸시하여 실시간 Vercel 배포(`https://academy-attendance-nu.vercel.app/`)에 반영해야 합니다.
- 수정 완료 후 배포가 성공적으로 완료되었는지 Vercel 상태를 점검해야 합니다.

## 2. 브라우저 실시간 동작 및 화면 이중 검증
- 수정을 마친 후, 실제 배포된 사이트(`https://academy-attendance-nu.vercel.app/`)에 직접 접속하여 브라우저 환경에서 아래 요소들을 눈으로 보고 직접 조작해 보아야 합니다.
  1. **헤더 행 고정**: 스크롤을 끝까지 내렸을 때 첫 번째 행 제목(학년, 이름 등)이 위에 고정되어 있는지 스크롤을 움직이며 점검할 것.
  2. **CSS 간섭 체크**: HTML/JS 렌더러에 설정된 인라인 스타일(`style="..."`)이 CSS 파일(`style.css`)에 설정된 규칙을 덮어쓰거나 오작동을 유발하지 않는지 개발자 도구 등으로 점검할 것.
  3. **터치/마우스 너비 드래그**: PC 마우스뿐만 아니라 모바일/태블릿 터치 환경(`touchstart`, `touchmove`, `touchend`)에서도 열 너비 수동 조정이 부드럽게 작동하는지 검증할 것.
  4. **카테고리 컬러 및 라이브 렌더링**: [문해력교재관리] 등 카테고리 색상 지정 탭에서 글자 입력 시 즉시 연한 파스텔 색상이 입혀지며 실시간으로 정상 리렌더링되는지 확인할 것.

## 3. 데이터 송수신 규격 검증 (학년 등기화 등)
- 데이터 수정 시 화면에 표기되는 텍스트와 실제 데이터베이스/구글 시트로 전달되는 코드값의 정밀 매칭을 확인할 것.
  - 예: 화면 선택창(Dropdown)에 `"초등 1"`로 보일지라도, 내부 값(`value`)은 구글 시트와 연동되는 `"초1"`로 매핑되어 리셋이나 동기화 풀림 현상이 일어나지 않아야 함.
- 구글 시트 API 연동(`doGet`, `doPost`)과 원장앱 간의 데이터 입출력 시 404/405/no-cors 등 통신 장애 유무와 응답값의 정확성을 확인할 것.
